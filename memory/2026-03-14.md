## 🔧 技术调整

### 1. 问题修复 (用户反馈)
- ✅ **播放信息太靠下** → 改为悬浮层 (overlay)，播放时显示，暂停时隐藏
- ✅ **列表显示海报** → 添加 `poster` 字段支持，缩略图 35x35px
- ✅ **侧边栏可折叠** → 收起 125px / 展开 420px，带平滑过渡

### 2. CORS 方案变更
- **最初:** 尝试内置代理 (`/api/proxy/`)，使用 `getProxiedUrl()`
- **回滚:** 用户要求恢复直接访问，移除代理层
- **现状:** HLS.js 直接请求原始 URL，依赖浏览器跨域策略

### 3. UI 风格升级 (JW Player 参考)
参照 `chrome-extension://.../player.html` 的深色 skin：

**配色体系:**
- 主背景: `rgba(12,4,88,.8)` (#0c0458)
- 次背景: `rgba(7,1,66,.8)` (#070142)
- 强调色1: `#ffc914` (金黄)
- 强调色2: `#fa3823` (橙红)
- 边框/高亮: `#473aed` (紫蓝)
- 悬停: `rgba(26,8,204,.2)`

**布局参数:**
- 侧边栏收起: 125px (icon only)
- 侧边栏展开: 420px (full info)
- 列表项高度: 50px
- 缩略图尺寸: 35x35px (圆角, bg-[#00128f])
- 滚动条: 6px 宽, 紫色 thumb

**组件样式:**
- 添加按钮: 渐变背景 (金黄→橙红)
- 频道项: flex layout, 删除按钮橙红色
- 分组标题: 深紫底 + 金黄文字
- 播放信息层: 底部渐变 overlay (`from-[rgba(10,2,96,.8)]`)
- 状态徽章: 边框 + 渐变背景

## 🧠 设计灵感来源

JW Player 的 `player.min.css` 展示了专业 IPTV 客户端的典型模式：
- 极窄收起状态 (125px) 保持上下文可见

- 玻璃态 (glassmorphism) + 深紫渐变
- 滚动条与主题色一致

## ⚠️ 已知限制

- **生产 Tailwind**: 仍使用 CDN，需改为本地构建
- **CORS**: 直接访问外部 m3u8 可能失败，需代理方案
- **错误处理**: 网络/媒体错误恢复逻辑未完全测试
- **无障碍**: 缺少 ARIA 标签和键盘导航完整支持
