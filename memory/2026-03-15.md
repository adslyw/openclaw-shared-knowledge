### Critical Bug Fix: Data Persistence
**Problem**: Clicking sync button, data appeared to save but was lost after page refresh.

**Root Cause**: `server.js` hardcoded database path to `path.join(__dirname, 'm3u-player.db')`, which creates the database inside the container at `/app/m3u-player.db`. The Docker volume was mounted at `/data`, but the app wasn't using it. When container restarted, data was lost.

**Solution**: Modified `server.js` to read `DATABASE_PATH` environment variable and fallback to default:

```javascript
const dbPath = process.env.DATABASE_PATH || path.join(__dirname, 'm3u-player.db');
const dbDir = path.dirname(dbPath);
if (!require('fs').existsSync(dbDir)) {
    require('fs').mkdirSync(dbDir, { recursive: true });
}
const db = new Database(dbPath);
```

**Verification**:
- ✅ Database file now created at `/data/m3u-player.db` inside container
- ✅ Host `./data/` directory contains persistent database files
- ✅ API sync/test confirmed data survives container restart
- ✅ Added directory creation to handle empty volume on first run
